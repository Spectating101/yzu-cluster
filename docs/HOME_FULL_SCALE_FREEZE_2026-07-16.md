# Research Drive — Home Full-Scale Freeze

**Status:** FROZEN VISUAL / INTERACTION AUTHORITY  
**Date:** 2026-07-16  
**Scope:** `Home` desktop composition, scaling limits, data honesty, and right-rail ownership  
**Current authority:** Visual Iteration 10 — Re-entry Desk  
**Freeze posture:** composition is closed; implementation, responsive convergence, and pixel polish must preserve this page model

This appendix preserves the approved Home composition reached after ten visual iterations.

The full-scale CLI wireframe below is implementation authority, not an illustrative sketch.

Home is not where sustained research happens.

```text
A NORMAL RESEARCH DRIVE SESSION

HOME
↓
DISCOVER
↓
LIBRARY
↓
SYNTHESIS
↓
LIBRARY
↓
DISCOVER
↓
...
```

The user may not revisit Home during a session. That is acceptable.

Home exists to reconstruct the desk quickly, surface useful evidence suggestions, expose the most relevant bounded resource headroom, and show durable recent machine consequences before the researcher returns to the owning work surface.

---

## 1. Frozen Home thesis

```text
PICK UP

where was the resumable work?

        +

RESOURCE HEADROOM

which 1–2 bounded capacities are worth remembering?

        ↓

RECOMMENDED EVIDENCE

what grounded evidence suggestion may be useful next?

        ↓

RECENT TRAIL

what durable machine consequence materially happened?
```

The four functions are binding:

```text
PICK UP
=
recover exact resumable work state

RESOURCE HEADROOM
=
glance at authoritative bounded capacity

RECOMMENDED EVIDENCE
=
surface grounded evidence suggestions with an exact Discover handoff

RECENT TRAIL
=
show material durable consequences, not browser history
```

Home is not:

```text
command centre
metrics dashboard
worker monitor
catalogue
job dashboard
generic chat landing page
activity feed
session analytics surface
news feed
research-method recommender
```

Do not show pseudo-personal telemetry such as:

```text
You were last here yesterday at 23:41.
```

Home may know resumable state. It must not make login chronology or inferred cognition the content experience.


## 4. Pick Up

`PICK UP` is the dominant Home territory.

It answers:

```text
WHERE IS THE EXACT WORK I CAN RESUME?
```

It does not answer:

```text
WHAT SHOULD I DO WITH MY LIFE?
WHAT WAS I THINKING?
WHEN DID I LAST LOG IN?
```

The target resume concept is a typed resume point:

```text
resume_point

kind
object_id
title
page
state_summary
location
last_interacted_at
```

`last_interacted_at` may help rank resume points. It is not normal user-facing Home copy.

Primary resume grammar:

```text
OBJECT TITLE

short exact state summary
material state change or unresolved state, when authoritative

OWNING PAGE / LOCATION                         [ CONTINUE ]
```

Example:

```text
PRIVATE USDT TRANSACTIONS

Source relationship review
BigQuery correspondence established;
equivalence remains unresolved.

LIBRARY / RAW EVIDENCE                         [ CONTINUE ]
```

The centre should not duplicate the full Detail rail.

```text
CENTRE
=
OBJECT + RESUMABLE STATE

DETAIL
=
STRUCTURED TRUTH + CURRENT DECISION
```

For a Home resume point selecting `library_asset`, the rail must use the same exact asset identity and the normal Library asset Detail / Ask contract.

Secondary resume grammar is one compact line:

```text
STABLECOIN EVENT STUDY PANEL
Input readiness · 1 unresolved gap · SYNTHESIS        Continue →
```

No third resume point is shown.

## 5. Resource Headroom

Resource Headroom is a glance surface.

It is not a miniature Resources page.

Frozen slot model:

```text
SLOT 1
USER-PINNED RESOURCE

SLOT 2
SYSTEM-RANKED MATERIAL / SIGNIFICANT RESOURCE
```

The first slot represents personal operational memory.

The second slot represents the capacity currently most worth seeing.

Ranking may consider:

```text
warning / degraded state
lowest authoritative headroom
high consumption
operational significance
active research dependency
```

The ranking explanation does not need to be rendered on Home.

A usage bar is permitted only when all of the following are authoritative:

```text
USED VALUE
+
REAL CAP / DENOMINATOR
+
CURRENT RESOURCE STATE
```

Therefore:

```text
2.1 TB / 5 TB
=
VALID BOUNDED BAR

1.8 TB / 2 TB
=
VALID BOUNDED BAR

BigQuery default max bytes billed per query
=
NOT A MONTHLY QUOTA BAR
```

Do not claim:

```text
renews tomorrow
resets Aug 1
12 days remaining
```

unless a real reset / renewal authority exists.

Resource row grammar:

```text
RESOURCE NAME                     PINNED when applicable
used / cap                         percent
bar
headroom
                                        Resources → | Check →
```

Maximum two resources.

The full usage picture remains owned by Resources.

## 6. Recommended Evidence

`RECOMMENDED EVIDENCE` is a grounded suggestion layer.

It is not a generic AI idea generator.

It may use durable profile / registry recommendation inputs such as:

```text
procurement_recommendations
starter recommendation authority where explicitly represented
known Library relationship
active research context where backed by durable product state
```

The current grounded profile model already distinguishes recommendations that are:

```text
in lab, not linked

or

not in lab yet
```

Home may surface at most two recommendations.

Normal row grammar:

```text
TITLE                                  STATE
one-line evidence description · concise recommendation reason      ACTION →
```

Examples:

```text
HISTORICAL USDT TRANSFERS              NOT IN LAB YET
Stablecoin transaction history · recommended for current research  Explore →

ISSUER RESERVE DISCLOSURES             NOT IN LAB YET
Issuer-level disclosure evidence for event timing                  Explore →
```

The recommendation row is not a card.

Do not add:

```text
confidence score
AI match percentage
model-generated novelty badge
fake urgency
fake source freshness
related method
research pulse
news taxonomy
```

Normal action ownership:

```text
NOT IN LAB YET
→ exact Discover handoff

IN LAB, NOT LINKED
→ Library / collection relationship action when truthful
```

If there is no grounded recommendation authority:

```text
RECOMMENDED EVIDENCE
DISAPPEARS
```

Do not fill the section with generic prompts.

## 7. Recent Trail

`RECENT TRAIL` is a compact consequence ledger.

It is not browser history.

Bad trail events:

```text
opened Library
previewed dataset
searched TWSE
clicked source
selected row
asked a question
```

Good trail events:

```text
scheduled collection completed
procurement completed
asset registered
refresh materially advanced
output registered
material failure recovered
researcher decision became available
```

Normal row grammar:

```text
CONSEQUENCE TYPE          OBJECT
one-line durable consequence                                      destination →
```

Examples:

```text
COLLECTION COMPLETED      MOPS GOVERNANCE PANEL
Evidence archived and registered.                                 Library →

REFRESH ADVANCED          GDELT ASIA
18 / 99 months collected.                                         History →

PROCUREMENT COMPLETED     HISTORICAL MARKET EVENTS
Registered in the evidence estate.                                Library →
```

Maximum three rows.

`REFRESH ADVANCED` must represent a material progress boundary or meaningful state change. Ordinary polling noise is not promoted.

```text
41% → 43%
=
NOT HOME TRAIL

18 / 99 months after a meaningful scheduled refresh boundary
=
MAY BE HOME TRAIL
```

`View all →` routes to the durable owning history / activity authority. Home does not own an expanded trail mode.

## 8. Detail / Ask contract

Home does not have a special generic Home inspector.

Selecting a typed resume point activates the exact underlying object.

Examples:

```text
Home resume point
kind = library_asset

→ Library asset Detail grammar
→ Library asset Ask grammar


Home resume point
kind = synthesis_recipe

→ Synthesis recipe Detail grammar
→ Synthesis recipe Ask grammar
```

Binding loop:

```text
HOME CENTRE
exact resume object + resumable state

        ↓

DETAIL
exact object meaning + structured truth + decision

        ↓

ASK
operate / reason over the same exact object

        ↓

DURABLE CONSEQUENCE

        ↓

OWNING SURFACE / HOME PROJECTION
state becomes visible
```

Do not duplicate the rail's verification, provenance, source-chain, or detailed readiness modules inside the Pick Up block.

The Home rail footer may expose the exact resume action and one contextual Ask action.

## 9. Bounded state behavior

The frozen composition must degrade without becoming a different page.

### No primary resume point

```text
PICK UP

NO RESUMABLE THREAD
No exact working context is currently preserved.

RESOURCE HEADROOM
remains in the top band when authoritative
```

Do not invent a research-intention hero or generic `What are you investigating?` input to fill the space.

### One resume point only

Show the primary resume point.

Do not promote a recommendation into the secondary resume slot.

### No pinned resource

Slot 1 may use the most significant authoritative resource until a pin exists.

Do not label it `PINNED`.

### Fewer than two bounded resources

Show only authoritative rows.

No fake placeholder bars.

### No grounded recommendations

Remove `RECOMMENDED EVIDENCE` content.

Preserve the page rhythm; do not inflate Recent Trail into a giant feed.

### No trail events

Show:

```text
RECENT TRAIL

No material machine consequences to show.
```

Do not substitute browser activity.

### Unbound profile

Recommendations requiring profile recommendation authority disappear.

Home must not silently use the pilot professor profile as the user's recommendation context.
