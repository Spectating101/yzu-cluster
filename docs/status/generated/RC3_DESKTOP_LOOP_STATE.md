# RC3 desktop convergence loop

Current generation: `rc3-desktop-convergence-loop-5`

Product authority for this loop:

- desktop web is the acceptance surface;
- RC2.1 supplies rhythm, density, and immediate usability;
- RC3 supplies typed research state, lifecycle truth, approval boundaries, and backend integration;
- faculty-facing records are default product state;
- validation and operator records remain available only through explicit technical views;
- live comparison must match the exact watched-branch SHA before visual acceptance.

Accepted desktop convergence through the previous checkpoint:

- Home ranks query-ready research assets above procurement receipts and keeps its context signals readable;
- Library defaults to the faculty-facing estate and exposes technical records explicitly;
- Synthesis defaults to substantive durable threads and develops drafts in place;
- Discover starts from held evidence, then distinguishes supported source routes and controlled acquisition;
- Resources separates actionable failures and approvals from raw capacity observations;
- the global header distinguishes raw registry records from faculty-facing holdings.

Loop 5 visual hypothesis under validation:

- the research canvas must dominate the desktop shell;
- sidebar and inspector are orientation/support surfaces, not equal product columns;
- Synthesis uses two internal columns because Composer is the persistent inspector mode;
- shared typography, spacing, and chrome should read as one authored research workbench.
