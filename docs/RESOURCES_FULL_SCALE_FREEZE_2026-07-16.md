# Research Drive — Resources Full-Scale Freeze

**Status:** FROZEN VISUAL / INTERACTION AUTHORITY  
**Date:** 2026-07-16  
**Scope:** `Resources` desktop composition, source/capacity authority, expenditure history, method visualization, state family, scaling, and Detail / Ask ownership  
**Current authority:** Visual Iteration 05  
**Freeze posture:** composition is closed; implementation, responsive convergence, accessibility, and pixel polish must preserve this page model

This appendix preserves the approved Resources composition reached after the audit-derived visual convergence pass.

The full-scale CLI wireframes below are implementation authority, not illustrative sketches.

Resources answers three distinct questions:

```text
SOURCES
What can the lab use now?

USAGE
What resources did the lab actually spend, when, and on what research object?

METHOD
How does evidence move through the machine?
```

The permanent Resources tabs remain exactly:

```text
Sources | Usage | Method
```

Resources is not a worker dashboard, generic activity browser, job table, infrastructure console, storage-only page, integration catalogue, billing centre, or architecture documentation surface.

---

## 1. Frozen Resources thesis

```text
CAPABILITY + CURRENT HEADROOM
                ↓
             SOURCES

RECORDED RESOURCE EXPENDITURE
                ↓
              USAGE

EVIDENCE-MOVEMENT METHOD
                ↓
              METHOD
```

The three tabs must not collapse into one generic operations view.

```text
SOURCES ≠ expenditure history
USAGE   ≠ browser / app activity
METHOD  ≠ worker dashboard
```

## 2.2 Frozen Sources composition

```text
CAPACITY & ACCESS

three paired rows maximum

STORAGE
Vault | Bulk cache

ACCESS
Working disk | Collector identities

EXECUTION
Connected hosts | Parallel capacity

────────────────────────

SOURCE CAPABILITIES

Licensed / institutional
Public market & filings
Research & open data
```

The capacity band is fixed.

The source-capability ledger may scroll inside the centre when necessary.

The application page itself and the right rail remain viewport-bound.

## 2.3 Progress-bar honesty

A progress bar appears only when all three are authoritative:

```text
CURRENT NUMERATOR
+
REAL DENOMINATOR
+
CURRENT STATE
```

Valid bar examples include:

```text
GDrive vault           used / total capacity
USB bulk cache         used / total capacity
Working disk           reported used percentage
Collector identities   configured / total profiles
Connected hosts        joined / total hosts
Parallel capacity      active / maximum parallel slots
```

Do not create percentage bars for:

```text
staging free space without total capacity
BigQuery period use against a per-query guard
Tavily period calls against a per-request guard
Ask turns without an allowance
source probes without an allowance
```

## 2.4 Source authority vocabulary

```text
OBSERVED
Current telemetry supports the access claim.

ROUTE DEFINED
A durable capability / route definition exists, but current successful access is not established.

CONDITIONAL
A source-specific probe, entitlement, approval, or other condition remains.

UNAVAILABLE
Current explicit evidence says the route cannot be used.

NOT CHECKED
Neither current positive nor current negative access authority exists.
```

A manifest declaration is not live provider health.

```text
ROUTE EXISTS
≠
ACCESS VERIFIED
```

The UI must not default provider rows to `Ready`, `Healthy`, or `Available` merely because the manifest contains them.

## 2.5 Source count honesty

Do not display a single authoritative provider count until the meaning of the manifest-visible count and rollup `source_count` is reconciled.

Acceptable interim labels:

```text
DECLARED
KEY ROUTES
CAPABILITY ESTATE
```

Not acceptable:

```text
14 LIVE SOURCES
10 VERIFIED SOURCES
```

without matching authority.

## 3.2 Frozen Usage hierarchy

```text
PERIOD TOTALS
↓
SPEND HISTORY
↓
PERIOD ATTRIBUTION
```

Usage is an expenditure ledger.

Every primary history row must answer:

```text
WHEN?
WHAT RESOURCE?
HOW MUCH?
FOR WHAT RESEARCH OBJECT?
WHAT RECORDED OUTCOME?
```

A history row may omit the research object only when the backend does not provide durable attribution; that absence must remain visible rather than being filled by inference.

## 3.3 Usage inclusion rule

Include only materially meaningful resource expenditure such as:

```text
BigQuery billed bytes
web-discovery calls
Ask / model turns
source probes
other explicitly metered resource consumption
```

Do not include:

```text
page navigation
opened Preview
selected a row
worker heartbeat
polling
ordinary job state changes
approval events without resource expenditure
generic unmetered app activity
```

The former `Activity` tab, activity-category browser, review queue, and generic run log are rejected Resources composition.

## 3.4 Attribution bars

Attribution bars may be used only when every segment shares the same recorded period denominator.

Valid example:

```text
USDT transfer panel      4.2 / 12.4 GiB
GDELT Asia crosswalk     3.1 / 12.4 GiB
Other recorded use       5.1 / 12.4 GiB
```

The UI must not imply complete object-level attribution when some resource events lack durable research-object identity.

```text

```

For Resources:

```text
CENTRE
selected resource / expenditure / method node
+
its comparative page context
+
its current visible state

DETAIL
exact meaning
current authority
known constraints

ASK
questions and supported operations scoped to the exact selected object
```

The centre does not duplicate the full rail explanation.

The rail remains viewport-bound:

```text
RAIL HEIGHT = APP VIEWPORT

FIXED IDENTITY / STATE
BOUNDED INTERNAL SCROLL
STICKY ACTION FOOTER
```

The page never grows because of rail content.

```text
CURRENT STATE UNAVAILABLE

ROUTE DEFINED

```

## 6.3 Usage · no recorded expenditure

```text
NO RECORDED RESOURCE USE

No metered resource expenditure was recorded
for Jun 1 — Jun 30, 2026.

Current capacity and access remain under Sources.
```

## 6.4 Method · no active bounded progress

```text
CURRENT METHOD PROGRESS

No bounded harvest or coverage operation
is currently reporting progress.

The method map remains available.
```

## 6.5 Partial authority

```text
ROUTE DEFINED
Current successful access not established.

NOT CHECKED
No current access result.

CONDITIONAL
A probe, entitlement, or review remains.
```

Never convert partial authority into generic `Healthy` or `Ready` copy.
